## Step-by-Step Bootstrap

### Phase 1 — Kubernetes Cluster

Provision a Kubernetes 1.30+ cluster (bare-metal, vSphere, or cloud):

```bash
# Minimum recommended node pools:
# - 3× control-plane: 4 vCPU / 8 GB RAM
# - 4× worker-general: 8 vCPU / 16 GB RAM / 200 GB NVMe
# - 1× worker-gpu: 8 vCPU / 32 GB RAM + NVIDIA GPU  (for Ollama)
# - 2× load-balancer: 2 vCPU / 4 GB RAM (labelled node-role=load-balancer)

# Label GPU node
kubectl label node <gpu-node-name> accelerator=gpu

# Label load-balancer nodes
kubectl label node <lb-node-1> node-role=load-balancer
kubectl label node <lb-node-2> node-role=load-balancer

# Verify cluster
kubectl get nodes -o wide
```

### Phase 2 — Clone GitOps Repository

```bash
git clone https://gitlab.kiruz.com/kiruz/e-commerceStack.git
cd e-commerceStack
```

### Phase 3 — Run Bootstrap Script

The bootstrap script installs all operators, generates secrets, and registers all apps with ArgoCD in the correct order. You can run all steps, or step by step:

```bash
# Make scripts executable
chmod +x scripts/*.sh

# Run full bootstrap (recommended for first install)
./scripts/bootstrap.sh all

# --- OR run individual steps ---
./scripts/bootstrap.sh 1   # Namespaces + ResourceQuotas
./scripts/bootstrap.sh 2   # Operators (cert-manager, nginx, Longhorn, CNPG, ESO)
./scripts/bootstrap.sh 3   # Generate/push secrets to Vault
./scripts/bootstrap.sh 4   # Install ArgoCD
./scripts/bootstrap.sh 5   # Register all ArgoCD apps
./scripts/bootstrap.sh 6   # Apply NetworkPolicies
./scripts/bootstrap.sh 7   # Install Velero + backup schedules
./scripts/bootstrap.sh 8   # Apply alert rules + Grafana dashboards
./scripts/bootstrap.sh 9   # Verify
```

### Phase 4 — Manual Steps (Post-Bootstrap)

#### 4.1 — Configure Cloudflare DNS

After `ingress-nginx` has a public IP (from MetalLB or cloud LB):

```bash
# Get the ingress IP
kubectl get svc -n ingress-nginx ingress-nginx-controller -o jsonpath='{.status.loadBalancer.ingress[0].ip}'

# ExternalDNS will automatically create A records for all Ingress objects.
# Verify in Cloudflare dashboard that the following records exist:
#   api.kiruz.com       → <ingress-ip>  (proxied)
#   shop.kiruz.com      → <ingress-ip>  (proxied)
#   auth.kiruz.com      → <ingress-ip>  (proxied)
#   erp.kiruz.com       → <ingress-ip>  (proxied)
#   argocd.kiruz.com    → <ingress-ip>  (NOT proxied — direct)
#   vpn.kiruz.com       → <wireguard-ip> (NOT proxied — direct)
```

#### 4.2 — TLS Certificates

cert-manager will automatically provision Let's Encrypt certificates for all Ingress objects with the annotation `cert-manager.io/cluster-issuer: letsencrypt-prod`.

First, create the ClusterIssuer:

```bash
cat <<EOF | kubectl apply -f -
apiVersion: cert-manager.io/v1
kind: ClusterIssuer
metadata:
  name: letsencrypt-prod
spec:
  acme:
    server: https://acme-v02.api.letsencrypt.org/directory
    email: platform@kiruz.com
    privateKeySecretRef:
      name: letsencrypt-prod
    solvers:
      - dns01:
          cloudflare:
            apiTokenSecretRef:
              name: cloudflare-api-token
              key: api-token
EOF
```

#### 4.3 — Keycloak Realm Configuration

```bash
# Port-forward to Keycloak
kubectl port-forward -n identity-security svc/keycloak 8080:8080 &

# Import the Kiruz realm (if not auto-imported)
curl -X POST http://localhost:8080/auth/realms/master/protocol/openid-connect/token \
  -d "client_id=admin-cli" \
  -d "username=admin" \
  -d "password=$(kubectl get secret keycloak-admin-secret -n identity-security -o jsonpath='{.data.admin-password}' | base64 -d)" \
  -d "grant_type=password" | jq .access_token

# Create OIDC clients for: saleor, argocd, grafana, n8n, vaultwarden
# (Use Keycloak Admin Console at http://localhost:8080/admin)
```

#### 4.4 — WireGuard Warehouse Connection

```bash
# On warehouse nodes, configure WireGuard peer config
# (generated by generate-secrets.sh, stored in Vault at secret/wireguard)

# Retrieve peer configs
vault kv get -format=json secret/wireguard | jq .data

# Deploy the WireGuard config on each warehouse node
# The K8s WireGuard pod handles the SD-WAN tunnel automatically
kubectl logs -n identity-security deploy/wireguard
```

#### 4.5 — MinIO Bucket Setup

```bash
# Port-forward MinIO
kubectl port-forward -n storage svc/minio 9001:9001 &

# Configure mc (MinIO client)
mc alias set kiruz http://localhost:9001 admin <password>

# Buckets are auto-created by the MinIO chart (defaultBuckets),
# but set lifecycle policies manually:
mc ilm add kiruz/saleor-media --expiry-days 3650  # keep media forever
mc ilm add kiruz/loki-chunks --expiry-days 30      # Loki retention
mc ilm add kiruz/velero-backups --expiry-days 90   # Velero retention
```

#### 4.6 — Saleor Initial Setup

```bash
# Run Saleor initial superuser creation
kubectl exec -n core-ecommerce deploy/saleor -c saleor -- \
  python manage.py createsuperuser --noinput \
  --email admin@kiruz.com

# Populate seed data (optional)
kubectl exec -n core-ecommerce deploy/saleor -c saleor -- \
  python manage.py populatedb --createsuperuser
```

#### 4.7 — Odoo Initial Setup

```bash
# Initialize Odoo database
kubectl exec -n core-ecommerce deploy/odoo-wms -- \
  odoo --init=all --database=odoo --stop-after-init

# Enable WMS modules
kubectl exec -n core-ecommerce deploy/odoo-wms -- \
  odoo --update=stock,stock_picking_batch,delivery --database=odoo --stop-after-init
```

#### 4.8 — MQTT Robot Authentication

```bash
# The Mosquitto password file is managed as a K8s Secret
# Add a new robot client:
kubectl exec -n warehouse-robots deploy/mosquitto -- \
  mosquitto_passwd -b /etc/mosquitto/passwd amr-004 "$(gen_pass 24)"

# Reload Mosquitto config
kubectl exec -n warehouse-robots deploy/mosquitto -- \
  kill -HUP 1
```

## GitOps Workflow (Day-2 Operations)

### Updating a Service

```bash
# 1. Create a feature branch
git checkout -b update/saleor-3.21

# 2. Update the image tag in values.yaml
#    apps/core-ecommerce/saleor/values.yaml → image.tag: "3.21"

# 3. Commit and push
git add apps/core-ecommerce/saleor/values.yaml
git commit -m "feat(saleor): bump to 3.21"
git push origin update/saleor-3.21

# 4. Create a Merge Request in GitLab
#    → CI pipeline runs: lint → build → smoke tests on staging
#    → After MR approval, merge to main
#    → ArgoCD detects change, syncs automatically within 3 minutes
```

### Rolling Back

```bash
# Option A: Git revert (preferred — keeps audit trail)
git revert HEAD --no-edit
git push origin main

# Option B: ArgoCD rollback to previous revision
argocd app rollback saleor --revision <revision-number>

# Option C: Velero restore (full namespace restore)
./scripts/backup-restore.sh restore <daily-full-backup-name> core-ecommerce
```

### Scaling a Service Manually

```bash
# ArgoCD will revert manual scaling — use Git instead:
# Edit apps/core-ecommerce/saleor/values.yaml → replicaCount: 4
# Git commit + ArgoCD auto-syncs within 3 minutes

# For HPA-managed deployments, adjust minReplicas/maxReplicas in values.yaml
```

## Security Operations

### Rotating Secrets

```bash
# 1. Generate new secret values
./scripts/generate-secrets.sh

# 2. ESO will automatically refresh Kubernetes Secrets within refreshInterval=1h
#    Force immediate refresh:
kubectl annotate externalsecret saleor-secret \
  -n core-ecommerce "force-sync=$(date +%s)" --overwrite

# 3. Restart pods to pick up new secrets
kubectl rollout restart deployment/saleor -n core-ecommerce
```

### Viewing Audit Logs

```bash
# Kubernetes audit logs (via Loki)
# In Grafana → Explore → Loki:
# {namespace="kube-system"} |= "audit"

# ArgoCD audit
argocd app history saleor

# Velero backup history
velero backup get -n velero
```

## Troubleshooting

| Symptom | Command | Resolution |
|---|---|---|
| ArgoCD app OutOfSync | `argocd app diff <app>` | Review diff, fix values, push |
| Pod CrashLoopBackOff | `kubectl logs <pod> -p` | Check previous container logs |
| Certificate not issued | `kubectl describe cert <name>` | Check ClusterIssuer, DNS |
| Robot not connecting | `kubectl logs deploy/mosquitto -n warehouse-robots` | Check MQTT ACL/password |
| DB connection refused | `kubectl exec deploy/saleor -- python manage.py dbshell` | Verify PgBouncer, NetworkPolicy |
| MinIO bucket missing | `mc ls kiruz/` | Re-run MinIO bucket setup step |
